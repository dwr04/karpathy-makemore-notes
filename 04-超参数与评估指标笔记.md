## 第一部分:超参数

## 1. 学习率(lr / alpha)—— 最重要的超参数

### 1.1 太大太小分别会怎样

| lr | 现象 | 你的实测(lr扫描) |
|---|---|---|
| 太小 | 收敛极慢,loss 慢慢挪 | lr=0.001 → loss 卡在 3.31 |
| 合适 | 稳步快速下降 | lr≈0.1 → loss 到 2.49 |
| 太大 | 震荡,越过最优点 | lr=0.5 → loss 2.58(开始不稳) |
| 爆炸 | loss 飙升/发散 | lr=1.0 → loss 反弹到 3.91 |

### 1.2 找学习率的经典技巧:LR range test(Karpathy 同款)

不靠猜,让 lr 在一次短训练里**从很小指数增长到很大**,画 loss vs lr,取下降最快那段:

```python
lre = torch.linspace(-3, 0, 1000)     # 指数从 10^-3 到 10^0
lrs = 10 ** lre
lossi = []
for i in range(1000):
    ix = torch.randint(0, Xtr.shape[0], (32,))
    ... forward/backward ...
    for p in parameters:
        p -= lrs[i] * p.grad          # 每步用递增的 lr
    lossi.append(loss.item())

plt.plot(lrs, lossi)                   # x 轴用 lr(或 log lr)
```

**怎么读这张图**:
- 左边平坦(lr 太小,没学动)→ 中间陡降(理想区)→ 右边反弹(lr 太大,发散)。
- **选"反弹点前面一点"的 lr**,不是选最低点(最低点已接近发散边缘)。

> 你的实测:最低点 lr≈0.095,所以 02-2/02-3 用 0.1 是对的。lr=1.0 直接发散。

### 1.3 学习率调度(schedule)

初期用大 lr 快速下降,后期用小 lr 精细收敛:

| 策略 | 做法 | 适用 |
|---|---|---|
| **阶梯衰减** | 到某步 lr÷10(02-3 用的:后 20% 从 0.1→0.01) | 简单有效,首选 |
| **余弦退火** | lr 按 cos 曲线平滑降到 0 | 现代训练常用,更平滑 |
| **warmup** | 前几百步 lr 从 0 线性升上来,再衰减 | Transformer/大 batch 必备,防开局爆炸 |

## 2. Batch size —— 噪声 vs 速度的权衡

### 2.1 本质:batch 是对"真梯度"的估计

全量数据算的梯度是"真梯度";mini-batch 只是它的**带噪声估计**。
batch 越大,估计越准,但每步越慢。你的实测:

| batch_size | 与真梯度的余弦相似度 | 相对误差 |
|---|---|---|
| 8 | 0.17 | 5.37 |
| 16 | 0.27 | 3.78 |
| 32 | 0.34 | 2.74 |
| 64 | 0.45 | 1.90 |
| 128 | 0.58 | 1.34 |
| 256 | 0.72 | 0.96 |
| 512 | 0.82 | 0.67 |

**规律:噪声按 ~1/√(batch_size) 下降**(batch 翻 4 倍,误差减半)。

### 2.2 关键权衡与经验

- **小 batch(8~32)**:梯度噪声大 → loss 曲线抖,但噪声能帮助**跳出局部极小、有正则效果**。每步快。
- **大 batch(256+)**:梯度准、loss 平滑、能吃满 GPU 并行,但**每步慢**,且泛化有时反而略差(著名的 "large-batch generalization gap")。
- **甜点通常 32~128**:噪声可接受,速度也够。你项目用 32 合理。
- **注意联动**:batch 变大后梯度更准、更"敢走",通常要**同步调大学习率**(经验:batch×k,lr×√k 或 ×k)。
- **BatchNorm 的坑**:batch 太小(<16)会导致批统计量不稳,BN 效果变差 —— 这是选 batch 时容易忽略的约束。

## 3. 其他关键超参数速查

| 超参数 | 作用 | 调大的后果 | 调小的后果 |
|---|---|---|---|
| `hidden_size` | 隐藏层宽度=模型容量 | 更强,但易过拟合+更慢 | 欠拟合 |
| `embedding_dim` | 每个字符的向量维度 | 表达更丰富 | 信息瓶颈 |
| `block_size` | 上下文长度(看几个字符) | 看得更远,输入维度线性增 | 上下文不足 |
| `iters` | 训练步数 | 学得更充分,可能过拟合 | 欠训练 |
| 正则强度(weight decay / dropout) | 抑制过拟合 | 欠拟合 | 过拟合不受控 |

**记忆**:前三个是**容量**类(欠拟合就调大),后两个和 lr/batch 是**训练动态**类。

## 4. 网格搜索 & 超参数搜索

### 4.1 三种搜索方式

| 方式 | 做法 | 优缺点 |
|---|---|---|
| **Grid Search 网格** | 每个超参列几个候选,笛卡尔积全跑 | 直观但组合爆炸:3参×5值=125次 |
| **Random Search 随机** | 在范围内随机采样 N 组 | **通常比网格更高效**(重要参数覆盖更密) |
| **Bayesian / Optuna** | 用已跑结果指导下一组(如 Optuna) | 最省算力,适合贵的实验 |

> 经验结论(Bergstra & Bengio 2012):**随机搜索普遍优于网格搜索**,
> 因为网格在不重要的参数上浪费了大量点,随机搜索对重要参数的采样更充分。

### 4.2 网格搜索代码骨架

```python
import itertools
grid = {
    'lr':      [0.05, 0.1, 0.2],
    'hidden':  [100, 200, 300],
    'embd':    [10, 20],
}
results = []
for lr, hidden, embd in itertools.product(*grid.values()):
    model = build(hidden, embd)
    train(model, lr, iters=5000)          # 每组只训短一点,省时间
    dev = eval_dev(model)                 # ★ 一定用 dev,绝不用 test
    results.append((dev, lr, hidden, embd))
    print(f"lr={lr} hidden={hidden} embd={embd} -> dev={dev:.4f}")

results.sort()
print("最佳:", results[0])                 # dev loss 最低的一组
```

### 4.3 搜索的三条纪律

1. **只看 dev,永远不碰 test**:test 是最后报告用的,搜索期间看了就等于泄漏。
2. **对数尺度搜连续参数**:lr、weight decay 这类要在 `[1e-4, 1e-1]` 上按 10 的幂采样,不是线性。
3. **先粗后细**:先大范围粗扫定位好区间,再在附近细扫。别一上来就密集网格。
4. **固定随机种子**:不然分不清是超参好还是运气好。

## 第二部分:评估指标大全(采集 → 展示 → 分析)

> 核心思想:**训练不是黑箱**。在循环里埋点采集,画出来,就能看懂网络内部发生了什么,
> 进而对症调参。下面按"采集什么 → 怎么画 → 说明什么 → 怎么用"组织。

## 5. 损失类指标

| 指标 | 采集 | 展示 | 用途 |
|---|---|---|---|
| **train loss(每步)** | `lossi.append(loss.item())` | `plt.plot(tensor(lossi).view(-1,100).mean(1))` 每100步取均值去噪 | 看收敛趋势、是否还在降 |
| **dev loss(每N步)** ★ | 每 500 步 `split_loss(Xdev)` 存入 `dev_lossi` | train/dev 两条线画一起 | **看何时开始分叉=过拟合起点** |
| **perplexity** | `exp(loss)` | 同上 | 更直观,可跨模型比较(2.21→ppl 9.1) |

**分析口诀**:
- train 和 dev 都高且贴合 → **欠拟合**,加容量/加训练。
- train 低、dev 明显更高(分叉)→ **过拟合**,加正则/减模型。
- 两条都平了不再降 → 收敛了,可以停或调 lr 再榨。

## 6. 激活分布指标(诊断饱和)

```python
# 每隔100步存一次隐藏层激活的快照
if i % 100 == 0:
    snapshot_a1[i] = a1.detach().cpu().numpy().copy()
```
- **展示**:`plt.hist(a1.flatten(), bins=50)` 直方图。
- **健康**:钟形集中在 0 附近;**病态**:大量堆在 ±1(tanh 饱和)。
- **你的实测**:02-2 饱和 23% → 加 BatchNorm 后 02-3 降到 6%。
- **用途**:饱和高 → 压 W1 初始化增益 / 加 BatchNorm。

## 7. update:data ratio(诊断学习率是否合适)

```python
ud_w1.append(torch.log10(lr * dW1.std() / (W1.std()+1e-8)).item())
```
- **展示**:`plt.plot(ud_w1); plt.axhline(-3)` 多条参数各画一线,配 -3 参考线。
- **含义**:这一步把参数改变了百分之几。**理想 log10 ≈ -3**(每步动 0.1%)。
- **读法**:高于 -3 → 更新太猛(lr 偏大/该参数尺度问题);低于 -3 → 学太慢。
- **你的实测**:W1/C 贴 -3(好),W2 常在 -1.7(偏高)→ 提示给 W2 单独小 lr 或换 Adam。
- **务必也监控** bngain/bnbias、以及分开记录 `grad.std()` 和 `weight.std()`(区分是分子还是分母的锅)。

## 8. 权重/梯度健康指标

| 指标 | 采集 | 说明 |
|---|---|---|
| 权重 std / 直方图 | `W1.std()` | 太大→将爆炸,太小→将消失 |
| 梯度 std | `dW1.std()` | 逐层递减是正常的(见梯度流笔记);突然变大=爆炸前兆 |
| **NaN/Inf 断言** | `assert torch.isfinite(dW1).all()` | 爆炸第一时间报错,不用等训练完 |
| 梯度范数 | `total_norm = sqrt(Σ‖g‖²)` | 做梯度裁剪时的判据,飙升=爆炸 |

## 9. 定性指标(数字之外)

| 指标 | 做法 | 为什么必要 |
|---|---|---|
| **定期采样生成** | 每 1~2k 步用当前模型生成几个名字打印 | loss 好 ≠ 生成质量好,要肉眼看像不像 |
| **训练集重复率** | 生成一批,统计有多少和 names.txt 完全重合 | 检测"记忆"而非"泛化",模型变大后必看 |
| **单步耗时** | `time.time()` 差值 | 放大模型/网格搜索前掂量时间成本 |

## 10. 一份"训练监控面板"该长什么样

一次训练结束,建议同屏展示这 4~6 张图,一眼看全:

```
┌─────────────────┬─────────────────┐
│ ① train/dev loss │ ② perplexity     │   ← 收敛 & 过拟合
├─────────────────┼─────────────────┤
│ ③ a1 激活直方图  │ ④ update:data ratio │  ← 饱和 & 学习率
├─────────────────┼─────────────────┤
│ ⑤ 各层梯度std曲线 │ ⑥ 定期采样文本   │   ← 梯度健康 & 定性
└─────────────────┴─────────────────┘
```

**看板 → 决策 映射**:
- ①分叉 → 加正则;①都高 → 加容量。
- ③饱和 → 调初始化/加 BN;④偏离-3 → 调 lr。
- ⑤梯度飙升 → 梯度裁剪;⑥不像名字 → 继续训/查数据。

## 11. 一句话总结

- **学习率**最关键:用 LR range test 找(你实测最优≈0.1,lr=1 发散),后期用衰减/退火。
- **batch size**是噪声↔速度权衡:噪声按 1/√bs 降,甜点 32~128,变大要同步升 lr,BN 需 bs≥16。
- **容量类超参**(hidden/embd/block)欠拟合才调大;**搜索用随机≥网格**,且**只看 dev**、对数尺度、先粗后细。
- **评估的精髓是埋点**:loss(train/dev)、激活直方图、update:data、梯度std、定性采样 —— 采集→展示→对症下药。
- **训练不是黑箱**:每个指标都对应一个可执行的调参动作。
