# 数据流 & 梯度流:一份数据穿过网络时,均值/方差怎么变

> 目标:建立"数值直觉"。前向时数据的 mean/std 每经过一层怎么变,tanh 饱不饱和,
> logits 大不大;反向时梯度大概什么量级。所有数字都是用你的架构
> (embd=10, block=3, hidden=200, vocab=27, batch=1000)在 `names.txt` 上**实测**的。
> 配套:`01-MLP训练完整笔记.md`(全流程)、`06-embedding反向传播笔记.md`(demb→dC)

---

## 0. 一句话主线

> **好的初始化 = 让数据穿过每一层后,std 都稳定在 1 附近,不爆炸也不塌缩。**
> 一旦某层把 std 放大到 6,tanh 立刻饱和 65%,梯度就死了;
> 一旦某层把 std 压到 0.5,信号越传越弱,也学不动。

---

## 1. 起点:embedding C

```python
C = torch.randn((vocab_size, embedding_dim))   # 标准正态,mean≈0, std≈1
emb_flat = C[Xb].reshape(m, block*embd)        # 查表+展平,只是搬运,不改分布
```

| 张量 | mean | std | 说明 |
|---|---|---|---|
| `C` | 0 | 1.0 | randn 的天然属性 |
| `emb_flat` | ≈0 | **≈1.1** | 查表和 reshape 不做运算,分布基本原样保留 |

**结论**:进入第一个线性层之前,数据是 **中心0、std≈1** 的漂亮状态。
接下来 W1 会不会破坏它,全看 W1 怎么初始化。

---

## 2. 关键一步:经过 W1 后数据怎么变

### 2.1 方差传播公式(理解一切的钥匙)

对 `hprebn = emb_flat @ W1`,每个输出元素是 `fan_in` 个"输入×权重"的乘积之和。
独立零均值变量相加,方差相加:

```
Var(hprebn) ≈ fan_in × Var(W1) × Var(input)

std(hprebn) ≈ sqrt(fan_in) × std(W1) × std(input)
```

其中 `fan_in = embd*block = 30`,`sqrt(30) ≈ 5.48`。

**这就是问题根源**:输入 std≈1,但线性层会把它乘上 `sqrt(fan_in)≈5.48`!
如果 W1 的 std 是 1(naive),输出 std 就被放大约 5.5 倍。

### 2.2 实测对比:5 种 W1 初始化

| W1 初始化方式 | W1_std | → hprebn_std | → a1_std | tanh饱和% |
|---|---|---|---|---|
| **naive** `randn*1` | 0.99 | **5.79** 💥 | 0.93 | **64%** 大量死亡 |
| 过小 `*0.1` | 0.10 | 0.54 🥶 | 0.44 | 0% (但信号太弱) |
| LeCun `1/√fan_in`(仅前向) | 0.19 | 1.06 | 0.64 | 1.5% |
| Kaiming(relu) `√2/√fan_in` | 0.26 | 1.47 | 0.73 | 7% |
| **Kaiming(tanh)** `(5/3)/√fan_in` | 0.30 | **1.66** ✅ | 0.75 | 11% |

> `emb_flat` 基准 std = 1.09。看 hprebn_std 那一列:
> naive 直接冲到 5.79,Kaiming 稳在 1.66,过小塌到 0.54。

### 2.3 为什么 naive 是灾难(数据说话)

```
naive:  hprebn std = 5.79  →  tanh(±5.79 这种大数)几乎全是 ±1
        →  64% 的神经元 |a1|>0.99  →  这些位置 tanh 导数 = 1-a1² ≈ 0
        →  梯度传不回去,等于 2/3 的网络一出生就"脑死亡"
```

实测 naive 的 `1-a1²` 均值只有 **0.13**,Kaiming 是 **0.40** —— 差 3 倍,
意味着 naive 的梯度天然被 tanh 掐掉了 2/3。

### 2.4 三种改善方式的数学推导

目标:让 `std(hprebn) ≈ 1`(或略大以补偿 tanh)。由公式
`std(hprebn) = sqrt(fan_in) × std(W1) × std(input)`,令输入 std=1,反解 W1。

**先分清一个关键点:前向要稳,反向也要稳。**
一层既参与前向(输出 = W·输入,扇入 `fan_in`),也参与反向(梯度 = W^T·上游梯度,扇出 `fan_out`):
- **前向**方差稳:`fan_in × Var(W) = 1` ⟹ `Var(W) = 1/fan_in`
- **反向**方差稳:`fan_out × Var(W) = 1` ⟹ `Var(W) = 1/fan_out`

除非 `fan_in = fan_out`,这两个条件没法同时满足 → 于是有了不同流派:

**① LeCun** —— 只顾前向
```
Var(W) = 1/fan_in        ⟹   std(W) = 1/sqrt(fan_in)
```
只考虑扇入。实测 a1_std=0.64,偏小。
(注:很多资料/我早期版本把它叫成"Xavier",严格说这是 LeCun 初始化。)

**② Xavier / Glorot** —— 前向反向折中(你在别处读到的那个)
```
两约束取平均:  Var(W) = 2/(fan_in + fan_out)
std(W) = sqrt(2/(fan_in+fan_out)) = 1/sqrt((fan_in+fan_out)/2)
```
把 `1/fan_in` 和 `1/fan_out` 折中,让前向和反向的方差都尽量稳。
PyTorch 的 `nn.init.xavier_normal_` 用的就是这个。gain=1(对称激活)。

**③ Kaiming / He** —— 只顾前向,但按激活函数补偿收缩
```
std(W) = gain / sqrt(fan_in)
```
激活函数会衰减信号(tanh、relu 都把一部分压掉),用 `gain` 补回来:
- relu:一半输出被置0,方差减半 → `gain = sqrt(2)`
- **tanh:PyTorch 给的经验增益 `gain = 5/3 ≈ 1.667`**

**④ 你代码里用的就是 ③ 的 tanh 版**
```python
W1 = torch.randn((fan_in, hidden)) * (5/3)/fan_in**0.5
```
实测 hprebn_std=1.66(故意略大于1,因为 tanh 马上会把它压回 0.75),
饱和率 11%,是这几种里最健康的平衡点。

> **为什么现代主流是"只除 fan_in"的 Kaiming,而不是折中的 Xavier?**
> Xavier 那半"顾反向"的约束,如今交给 **BatchNorm/LayerNorm/残差连接** 去兜了,
> 所以只按 fan_in 缩放的 Kaiming 反而更常用。你的 makemore 隐藏层
> `fan_in=30, fan_out=200` 差很大,理论上 Xavier 更稳,但配了 Kaiming+(后来的)BatchNorm
> 一样跑得很好 —— "只除 fan_in" 不是错,是被实践验证过的另一条路线。

### 2.5 公式验证(理论 vs 实测)

```
预测: std(hprebn) = sqrt(30) × 0.304 × 1.12 = 1.867
实测: std(hprebn) = 1.66 ~ 1.88
```
基本吻合(差异来自 emb 不完全独立、有限样本)。**公式是可靠的直觉工具。**

## 3. 经过 tanh 后:std 一定会缩小

tanh 是压缩函数(斜率≤1,且两端饱和),所以**无论输入如何,输出 std 必然 < 输入 std**:

| | hprebn_std (tanh前) | a1_std (tanh后) | 变化 |
|---|---|---|---|
| Kaiming(tanh) | 1.66 | 0.75 | 缩到约 45% |
| naive | 5.79 | 0.93 | 大数被压平到 ±1,std 反而"虚高" |

- **mean**:tanh 是奇函数,输入近似对称 → 输出 mean 依然 ≈0(实测 -0.03)。
- **健康信号**:a1 的 std 在 **0.7 左右**、饱和率 **10% 上下** 是好状态。
  太高(>0.9 且饱和>50%)=挤在两端;太低(<0.5)=挤在中间线性区,白瞎了非线性。
- 这也解释了 `MLP训练完整笔记` 里的直方图:02-2 训完饱和 23%,加 BatchNorm 后降到 6%。

## 4. 经过 W2 后:logits 要"小而平"

```python
W2 = torch.randn((hidden, vocab_size)) * 0.01   # 故意压到极小
logits = a1 @ W2 + b2
```

实测 `logits_std ≈ 0.10`,`logits_mean ≈ 0`。

**为什么 W2 用 0.01 而不是 Kaiming**:输出层的诉求和隐藏层相反——
这里**不希望**方差=1,而希望 logits **接近全 0**,让 softmax 输出接近均匀分布:

```
logits ≈ 0  →  softmax ≈ 均匀 1/27  →  初始 loss ≈ -ln(1/27) = 3.296
```

实测初始 loss = 3.30,和理论值 3.296 几乎相等 ✅。
如果 W2 不压小,logits std 会很大,某些字符初始概率畸高,loss 冲到几十,
前几百步全浪费在"把错误的自信掰回来"(loss 曲线开头那根曲棍球柄)。

> **两个线性层,两种哲学**:
> W1(隐藏层)要 **保方差**(Kaiming,std→1);
> W2(输出层)要 **压方差**(*0.01,logits→0)。

## 5. 前向数据流总览(Kaiming 版,一张表)

| 阶段 | mean | std | 关键点 |
|---|---|---|---|
| C / emb_flat | ~0 | 1.1 | 起点,漂亮 |
| **@W1** hprebn | ~0 | 1.7 | 乘了 √fan_in,靠 W1 缩放压住 |
| **tanh** a1 | ~0 | 0.75 | 必然缩小,饱和率~11% |
| **@W2** logits | ~0 | 0.10 | 故意压小 → 初始 loss≈3.3 |

一句话:**std 走了 1.1 → 1.7 → 0.75 → 0.10 的轨迹**,全程被控制在 O(1) 量级内,
没有任何一层爆炸或塌缩,梯度才能顺畅双向流动。

---

## 6. 反向:梯度大概什么量级

反向从 loss 出发,梯度**逐层变小**(每过一个 @W 就乘一个小权重矩阵)。实测(Kaiming):

| 梯度 | std 量级 | 说明 |
|---|---|---|
| `dlogits = (probs - onehot)/m` | ~1e-3 | 除以了 batch size m,天然很小 |
| `dW2` | ~1.3e-2 | = a1.T @ dlogits,累加 m 项所以比 dlogits 大 |
| `dz1 = da1*(1-a1²)` | 被 tanh 导数缩放 | **naive 时 (1-a1²)≈0.13 把梯度掐掉 6~7 成** |
| `dW1` | ~4.6e-4 | 又过了一层,更小 |
| `dC` | ~4.8e-4 | 最深处,最小 |

### 6.1 两条核心规律

**规律一:梯度越往输入层越小。**
每次 `dh_prev = dh @ W.T`,乘的是 std≈0.3 的权重矩阵,信号一路衰减。
所以 `dC (1e-4) < dW1 (1e-4) < dW2 (1e-2)`。这就是"梯度消失"的雏形——
层数越深越严重,也是为什么深网络更依赖 BatchNorm / 残差连接。

**规律二:初始化直接决定反向能不能活。**
对比 naive vs Kaiming 的 tanh 导数 `1-a1²`:
```
naive:    均值 0.13   →  dz1 只剩 13%,W1 和 C 几乎学不动
Kaiming:  均值 0.40   →  dz1 保留 40%,梯度能有效回传
```
**前向的饱和 = 反向的梯度死亡,是同一枚硬币的两面。**

### 6.2 为什么要看 update:data ratio 而不是看梯度绝对值

梯度绝对值小(1e-4)不代表学得慢,关键是**相对于参数自身的比例**:
```
实际步长占比 = lr × std(grad) / std(param)   →  理想 log10 ≈ -3 (每步动 0.1%)
```
所以 `MLP训练完整笔记` 第 8 节盯的是这个比值,而不是 dC 的绝对大小。
W2 的比值偏高(-1.7),正是因为它梯度大(1e-2)而自身权重小(0.01)。

## 7. BatchNorm 如何改写这套数据流

BatchNorm 插在 W1 和 tanh 之间,**强制**把 hprebn 拉回 mean=0, std=1:

```
无 BN:  hprebn 的分布好坏 = 全押在 W1 初始化上,调错就饱和
有 BN:  hprebn → (减均值/除标准差) → 无论 W1 怎样,进 tanh 前都是 N(0,1)
```

- **好处**:初始化不再那么敏感,饱和率天然被压住(02-3 实测 6%)。
- **代价**:同 batch 样本统计量耦合,引入噪声(轻微正则化),且训练/推理要用不同统计量
  (推理用 running 值,见全流程笔记第 6 节)。

**理解要点**:BatchNorm 本质是"每一层都主动把数据流拽回 std=1",
把第 2 节那个需要精心调 W1 才能达到的目标,变成了一个自动、可学习的操作。

---

## 8. 一句话总结

- **数据流主线**:std 走 `1.1 →(@W1)1.7 →(tanh)0.75 →(@W2)0.10`,全程 O(1),不爆不塌。
- **W1 初始化数学**:`std(hprebn)=√fan_in·std(W1)·std(input)`,令其≈1 ⟹ `std(W1)=gain/√fan_in`;tanh 的 gain=5/3。
- **naive 的死法**:W1*1 → hprebn std=5.8 → tanh 饱和 64% → 前向挤两端、反向梯度剩 13%,直接脑死亡。
- **两层两哲学**:隐藏层 W1 **保方差**(std→1),输出层 W2 **压方差**(logits→0,初始 loss≈3.3)。
- **梯度流规律**:越往输入越小(dC≈1e-4 < dW2≈1e-2);前向饱和 = 反向梯度死,是一回事。
- **BatchNorm**:把"手调 W1 让 std=1"自动化成每层强制归一化,初始化不再敏感。
